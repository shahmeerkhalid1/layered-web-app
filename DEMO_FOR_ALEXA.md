# Pilates Platform Demo for Alexa

## Opening Line

"Alexa, last time we discussed the Pilates platform as one organized place for instructors to manage their work. Today I want to show you the secure foundation working: login, registration, roles, admin control, and invite-only onboarding."

## Steps

### 1. Pull up the login page.

"This is the front door of the platform. Instructors and admins sign in with email and password, so the workspace is protected from the start."

Point out the clean login form.

"Before we add sensitive client notes, attendance, and class plans, we need to make sure only approved users can access the system."

### 2. Log in and show the main dashboard.

"This is the instructor dashboard. Right now the numbers are placeholders, but the structure is already prepared for the daily Pilates workflow."

Point at the dashboard cards: today's classes, total clients, exercise library, and plan templates.

"These cards will later connect to real scheduling, client, exercise, and planning data."

### 3. Show the admin area in the sidebar.

"The platform understands roles. A normal instructor gets the teaching workspace. An admin gets extra tools to manage the platform."

Click the Admin page.

"This admin overview gives You(Alexa) a simple control panel: total instructors, active users, inactive users, and pending invitations."

### 4. Open User Management.

"This is where You(Alexa) can manage users."

Point at the table, role column, status column, and action menu.

"You can search users, see whether someone is an admin or instructor, check if the account is active, view details, change roles, and activate or deactivate access."

### 5. Click "Invite user."

"Instead of letting anyone register, You(Alexa) can invite a specific instructor by email."

Choose a role in the invite form.

"You(Alexa) can decide whether the person joins as a normal instructor or as an admin. The system creates an invitation link tied to that email."

Point at the invite link after creation.

"If email is configured, this can be sent automatically. If not, the link can still be copied and shared manually."

### 6. Open the registration page or invitation link.

"The registration page checks whether signup is allowed. If the platform is invite-only, only invited users can create an account."

If using an invite link, point at the pre-filled email.

"This keeps onboarding controlled. Random users cannot just join the platform when invite-only mode is active."

### 7. Open Platform Settings.

"This setting controls whether registration is public or invite-only."

Toggle or point at the signup setting.

"For the early version, invite-only mode is safer. Alexa can test with selected instructors first, then open public signup later when ready."

## Closing

"So the main thing to see today is that this is not just a static design. The secure base is working: login, registration, admin roles, invite-only onboarding, user management, account activation, and platform settings."

"The next step is building the daily Pilates workflow on top of this foundation: exercise library, class plan templates, calendar, client profiles, attendance, session notes, and simple reports."

"The platform now has its front door, admin control, and dashboard structure ready. From here, we can safely add the core instructor features."  
  
Option for Deployment:

"For deployment: frontend on Vercel, backend on Render, Railway, or Fly.io."

"The frontend calls the backend API URL, and the backend connects to the hosted database."

Total time: ~5-10 minutes.
