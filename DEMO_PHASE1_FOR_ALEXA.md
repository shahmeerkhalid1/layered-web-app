# Pilates Platform — Demo for Alexa (Exercise Library & Class Plans)

## Overview — say first (about 30–45 seconds)

"**Last time** we covered the foundation: login, the dashboard, admin panel, inviting instructors, and controlled signup."

"**Since then** I finished the **exercise library** and the **class plan** side for instructors — with proper checks on the forms, the **layering system** for teaching steps and an optional **finisher**, clearer **listing pages** (folders, search, and actions together in one header on both screens), and everything **saved to the live server and database**. Class plans work as **reusable templates**: folders, search, class type and style filters, tags, sections, and pulling exercises from the library into a plan. The one piece still **not in the build yet** is **quick scheduling** — the shortcut to drop a plan straight onto the calendar in one go."

"**Today** I will walk you through **both** the exercise library and class plans so you can see how instructors move from saving moves to building a full session."

## Opening Line

"Alexa, you already saw the secure login and admin area. Today I am showing **two connected pieces**: the **exercise library** where instructors save and organise exercises, and **class plans** where those same exercises slot into reusable templates for a full class."

## Part 1 — Exercise Library

### 1. Open the Exercise Library from the menu.

"This is the instructor's list of exercises. They build each move once, then **reuse it inside class plans** — and later we will add the full calendar on top of that."

### 2. Show folders and search in the header.

"Everything they need to find exercises is at the top — no separate sidebar."

"They pick a folder from the **dropdown**  — Reformer, Mat, or whatever they created. The count shows how many exercises sit in that folder."

Point at the folder dropdown. Pick a folder so the count updates.

"**Rename**, **Delete**, and **New folder** sit right here when they need to manage folders."

Point at Rename / Delete and New folder.

### 3. Use search.

"When the list grows, they type a word — name or something in the description — and the list filters straight away."

Show typing in the search bar.

"If a folder and search shows nothing, they can **clear** search so they are not stuck."

### 4. Open an exercise or click to add a new one.

"Each exercise has a name, what it is for, starting position, and the teaching steps in order."

Point at the main blocks only. Do not read every field.

"The form walks them through what to fill in — no need to remember a special format."

### 5. Equipment, spinal movement, and chain type.

"**Equipment** uses simple checkboxes. They tick what applies. If they pick **None**, the other options switch off so the card stays tidy."

Point at equipment. Show None versus a few ticks.

"**Spinal movement** works the same way — tick what fits, or **None** if it does not apply."

Point at spinal movement.

"**Chain type** covers open chain, closed chain, or both. If they pick **Both**, the single-sided options turn off so the answer stays clear."

Point at chain type.

"Equipment, spinal movement, and chain type stay consistent in the library — and the **same exercises** show up when we **pick from the library** inside a class plan."

### 6. Show the steps (layers) and finisher.

"This is the layering system: Layer 1, Layer 2, and so on. They build intensity the way they already cue in the studio."

Point at Layer 1 and Layer 2.

"The **finisher** tick only appears on the **last** step. They use it when that step is the strong close. It is optional."

### 7. Show photos.

"They can add photos, drag to reorder, and tap one to view it large before class."

Point at thumbnails and the lightbox if you open it.

"Useful for remembering the shape or setup they want to teach."

### 8. Show the exercise detail page (read-only).

"This is the read-only view — it looks like clean, organised teaching notes"

Scroll a little.

## Part 2 — Class Plans

### 9. Open Class Plans from the menu.

"This is the **class plan library** — same idea as exercises: a list of **saved templates**."

### 10. Show the header — folders, search, and filters.

"The header matches what you saw on exercises: **New plan**, **New folder**, search, folder dropdown with counts, and filters for **class type**, **class style**, and **tags** so instructors can find the right template fast."

Point at the main controls. Optionally change a filter and clear filters.

### 11. Open a template or create one — sections and exercises.

"Think of a **template** as a class they might teach many times. They give it a name, set the basics like **type**, **style**, **how long it runs**, which **folder** it sits in, and any **tags** that help them find it later."

"Then they break the class into **sections** — whatever labels match how they teach, like a warm-up, the main work, and a cool-down — and they slot **exercises** under each part so the order matches how they would actually run the room."

"Most of the time they **pull moves from the exercise library** they already built. If a one-off variation only matters for this plan, they can still **add something new** right there without breaking the flow."

Point at a couple of sections and at least one exercise row. If you tweak something live, keep it quick and high level.

"When they **save**, it all sticks — they can come back whenever they want, run it again as-is, or open it up and adjust the template without rebuilding from scratch."

### 12. What is not in this build yet (say in one breath).

"**Quick scheduling** — the one-click **schedule this plan** style shortcut — is the bit I am still wiring next, before we move on to the **full calendar** view."

## Closing

"So today you saw **two working flows — exercise library and class plans**: instructors can run the **exercise library** end to end, and they can **build, filter, and save class plan templates** with sections and library exercises. **Quick scheduling** is the remaining gap on the class plan side; **calendar and full scheduling** come right after that."
